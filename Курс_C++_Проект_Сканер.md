## 🛑 САМОЕ ВАЖНОЕ ПРАВИЛО — прочитай вместе со взрослым

Сканер — это инструмент, который «прощупывает» компьютеры и порты. Применять его можно **ТОЛЬКО** к:

- 💻 **своему собственному компьютеру** (адрес `127.0.0.1` — это всегда «я сам»);
- 🏠 **своей домашней сети** — и желательно с ведома того взрослого, кто отвечает за домашний роутер;
- 🎯 **специальным учебным мишеням** (CTF, тренировочные лаборатории, которые прямо разрешают это).

❌ **Нельзя** сканировать чужие компьютеры, школьный/рабочий/публичный Wi-Fi, чужие сайты и сети — **даже «просто посмотреть»**. Во многих странах это нарушение закона.

## Часть 0. Что мы вообще делаем

🧠 Вспомним из файла «Под капотом»:
- **Порт** — это «дверь/квартира» на компьютере. У каждой сетевой программы своя дверь: у веб-сервера — `80` и `443`, у общего доступа Windows — `445`, и т.д.
- Если дверь **открыта** — значит, за ней работает какая-то программа и принимает гостей.

Что делают наши инструменты:
- **Сканер портов** — берёт один компьютер и **по очереди стучится во все двери** (порты), отмечая, какие открыты.
- **Сканер компьютеров** — берёт всю сеть и проверяет, **кто вообще дома** (какие IP-адреса отвечают).

🤖 Это крошечные версии знаменитой профессиональной программы **nmap**, которой пользуются все сетевые инженеры и пентестеры. Ты делаешь свой маленький nmap!

# Часть 1. Обзор: как собрать Qt C++ консольный проект 🛠

🤖 Короткая «шпаргалка-инструкция», как из ничего получить рабочую программу. Если делал файл про Qt — будет знакомо.

**Шаг 1. Создать проект.**
`File → New Project… → Application (Qt) → Qt Console Application → Choose…`
Имя: `Scanner`. **Build system: qmake** ⚠️. Kit: Desktop MinGW. → `Finish`.

**Шаг 2. Настроить `.pro`.**
Открой файл `Scanner.pro` и сделай его таким:

```pro
QT += core network        # ← network ОБЯЗАТЕЛЬНО: без него сети не будет
QT -= gui                 # графика не нужна

CONFIG += console c++17
CONFIG -= app_bundle      # (для macOS)

TEMPLATE = app
TARGET = Scanner
SOURCES += main.cpp
```

⚠️ Главная строчка — `QT += core network`. Забудешь `network` — компилятор не найдёт `QTcpSocket`. Это первая ошибка, на которой все спотыкаются.

**Шаг 3. Написать код** — в файле `main.cpp` (его наполним в Частях 3 и 4).

**Шаг 4. Собрать и запустить** — `Ctrl + B` (собрать), `Ctrl + R` (запустить). Внизу откроется консоль с результатом.

✅ **Правило Боба:** новый сетевой класс Qt не находится? Первым делом проверь строку `QT +=` в `.pro`.

# Часть 1.5. Разбор `.pro` под лупой 🔬

> Раздел для самых любопытных. Разберём наш `.pro` **построчно, максимально подробно** — чтобы ты понимал каждый значок, а не просто копировал.

## Сначала: что это за файл и кто такой qmake

🧠 Файл `.pro` — это **задание для программы qmake**. Сам он ничего не собирает. Цепочка такая:

```
Scanner.pro  ──читает──▶  qmake  ──создаёт──▶  Makefile  ──выполняет──▶  make
                                                                            │
                                              компилятор + линкер ◀─────────┘
                                                       │
                                                  Scanner.exe
```

🤖 То есть `.pro` — это **список пожеланий**: «вот такие модули мне нужны, такие настройки, такие файлы». qmake читает пожелания и пишет настоящую инструкцию по сборке (Makefile), а она уже запускает компилятор и линкер (помнишь цепочку из файла «Под капотом»?).

## Три главных значка-оператора

🧠 Каждую «переменную» (`QT`, `CONFIG`, `SOURCES`…) представь как **коробку со списком**. С коробкой можно делать три вещи:

| Оператор | Что делает | Аналогия |
|---|---|---|
| `=` | **выкинуть всё** и положить новое | «теперь в списке только это» |
| `+=` | **добавить** в список | «допиши ещё пункт» |
| `-=` | **убрать** из списка | «вычеркни пункт» |

Теперь каждая строка читается легко.

## Строка 1: `QT += core network`

🧠 **`QT`** — коробка, где перечислены **модули Qt** (части библиотеки), нужные проекту. **`+=`** — «**добавь** в список».

- **`core`** — **ядро Qt**: `QString`, `QObject`, сигналы/слоты, контейнеры, `QCoreApplication`, цикл событий. Фундамент, на него опирается почти всё.
- **`network`** — **сеть**: `QTcpSocket`, `QTcpServer`, `QUdpSocket`, `QHostInfo`. Ради него и пишем строку — без него сканера не будет.

🧠 Что делает qmake, когда ты добавляешь модуль:
1. Добавляет **пути к заголовкам** → чтобы `#include <QTcpSocket>` нашёлся.
2. Добавляет **библиотеку** модуля линкеру → привязка к `Qt6Network.dll`.
3. Добавляет нужные зависимости.

⚠️ Забыл `network` → две беды сразу: компилятор «не найден `QTcpSocket`», и/или линкер «undefined reference». Увидел такое — первым делом проверь `QT +=`.

🤖 `core` часто и так подключён по умолчанию — пишем для ясности; дубликат не вредит, qmake убирает повторы.

🕵️ **Реверс-связь:** из-за этой строки в готовом `.exe` появится **импорт `Qt6Network.dll`** — а импорты аналитик читает первым делом: «ага, работает с сетью».

## Строка 2: `QT -= gui`

🤖 **`-=`** — «**убери** из списка».

🧠 **`gui`** — модуль графики (`QGuiApplication`, `QImage`, окна, шрифты). Qt по умолчанию его подключает, но наш сканер **консольный** — ни окон, ни кнопок. Тащить графику, которой не пользуемся, незачем.

Что даёт: меньше лишних зависимостей, программа не линкуется с графикой, код «честнее». Оставишь `gui` — программа всё равно соберётся, просто потащит лишний груз (мы за правильность 😉).

🧠 Порядок важен: qmake читает сверху вниз. Строка 1 добавила `core`+`network`; в коробке был ещё `gui` по умолчанию; Строка 2 его вычёркивает. Итог: `core`, `network`.

## Строка 3: `CONFIG += console c++17`

🧠 **`CONFIG`** — коробка с **настройками сборки**. **`+=`** добавляет сразу два значения.

**`console`** — на Windows у программы есть невидимый флаг «подсистема»: **GUI** или **console**.
- Console-программа при запуске открывает чёрное окно, и туда идут `std::cout`, `qDebug`, `QTextStream(stdout)`.
- GUI-программа консоль не открывает — текстовый вывод может уйти «в никуда».
- `CONFIG += console` = «собери **консольную** программу», чтобы видеть результаты сканера. На macOS/Linux эффект почти нулевой, но строка безвредна.

**`c++17`** — выбор **версии языка C++**.
- Превращается во флаг компилятора `-std=c++17` (MinGW/GCC) или `/std:c++17` (MSVC).
- Включает современные возможности (например structured bindings `auto [a, b]`).
- Без неё компилятор возьмёт стандарт постарше, и часть современного кода **не соберётся**.

🤖 В `CONFIG` бывают и другие пункты: `release`/`debug`, `warn_on`, `c++20`, `app_bundle` (для macOS).

## Строка 4: `TEMPLATE = app`

🤖 Здесь **`=`**, а не `+=` — тип проекта один, список не нужен: «выкинь всё, тип — вот этот».

🧠 **`TEMPLATE`** говорит qmake, **что строим**:
- **`app`** — **приложение** (исполняемый `.exe`). Наш случай.
- `lib` — библиотека (`.dll`/`.so`).
- `subdirs` — «проект из проектов».
- `aux` — ничего не собирать.

🤖 `app` — значение по умолчанию, строку можно было бы и опустить, но явно — понятнее.

🕵️ **Связь с «Под капотом»:** `TEMPLATE` решает, что сделает **линкер** на последнем шаге — склеит в `.exe` (app) или в `.dll` (lib).

## Строка 5: `SOURCES += main.cpp`

🧠 **`SOURCES`** — коробка со списком файлов кода (`.cpp`) для компиляции. **`+=`** добавляет `main.cpp`.

🧠 Под капотом (привет, цепочка компиляции!):
- Каждый файл из `SOURCES` компилятор переводит в **объектный файл** (`.o`/`.obj`).
- Линкер склеивает все объектные файлы в программу.
- Файлов может быть много: `SOURCES += a.cpp b.cpp` или несколько строк.

⚠️ Классическая ошибка: создал новый `.cpp`, но **не добавил его в `SOURCES`** → он не компилируется, и непонятно, почему «не работает». (Через сам Qt Creator файлы добавляются в `SOURCES` автоматически.)

🤖 Родственные коробки в больших проектах:
- **`HEADERS +=`** — заголовки `.h` (особенно для классов с `Q_OBJECT` — их читает **moc**).
- **`FORMS +=`** — окна из Qt Designer (`.ui`).
- **`RESOURCES +=`** — картинки/иконки внутри программы (`.qrc`).

🤖 Почему `+=`, а не `=`? Безопасность: `+=` **дополняет** список и не стирает то, что добавили другие строки; `=` бы «выкинул всё».

## Соберём всё в одно предложение

🤖 Если прочитать `.pro` как просьбу к qmake:

> «Подключи модули **core** и **network**, графику **gui** убери. Собери **консольную** программу на **C++17**. Тип результата — **программа** `.exe`. Скомпилируй файл **main.cpp**.»

Полный путь до запуска:

```
.pro (пожелания) → qmake → Makefile → make → компилятор (main.cpp → main.o)
   → линкер (main.o + Qt6Core + Qt6Network → Scanner.exe) → 🎉 запуск
```

# Часть 2. Главный приём: «постучаться» в порт 🚪

🧠 Весь наш сканер держится на одной простой идее. В Qt есть класс `QTcpSocket` — это «телефонная трубка» для сети. Мы пробуем **дозвониться** на компьютер по конкретному порту:

```cpp
QTcpSocket socket;
socket.connectToHost("127.0.0.1", 80);   // звоним: адрес + дверь №80

if (socket.waitForConnected(200)) {       // ждём ответа до 200 миллисекунд
    // ДОЗВОНИЛИСЬ -> порт ОТКРЫТ
} else {
    // тишина/отказ -> порт закрыт
}
```

🤖 Вот и весь секрет:
- **дозвонились** (`waitForConnected` вернул `true`) → за дверью кто-то есть → **порт открыт**;
- **тишина** (вернул `false`) → дверь закрыта или компьютера нет.

🕵️ **Связь с «Под капотом»:** внутри `QTcpSocket` Qt вызывает те самые `socket`/`connect` из `ws2_32.dll` (Winsock). Удобная обёртка снаружи — знакомые функции внутри.

Теперь просто повторим этот «стук» в цикле много раз — и получим сканер.

# Часть 3. 🛠 Сканер портов (рабочая программа)

🧠 Берём один компьютер и стучимся во все двери подряд. Полный `main.cpp`:

```cpp
#include <QCoreApplication>
#include <QTcpSocket>
#include <QTextStream>

int main(int argc, char *argv[])
{
    QCoreApplication app(argc, argv);
    QTextStream out(stdout);            // удобный вывод в консоль (понимает QString)

    QString host = "127.0.0.1";        // ⚠️ только СВОЙ компьютер!
    int portFrom = 1;
    int portTo = 1024;

    out << "Scanning " << host << ", ports " << portFrom << ".." << portTo << "\n";
    out.flush();

    for (int port = portFrom; port <= portTo; ++port) {
        QTcpSocket socket;                    // «трубка» (сама закроется — RAII!)
        socket.connectToHost(host, port);     // стучимся в дверь №port

        if (socket.waitForConnected(200)) {   // ждём ответа до 200 мс
            out << "  port " << port << " — OPEN\n";
            out.flush();
            socket.disconnectFromHost();
        }
        // нет ответа -> порт закрыт, молча идём дальше
    }

    out << "Done.\n";
    return 0;
}
```

🤖 Разбор главного:
- `QTextStream out(stdout)` — наш «принтер». Удобнее `qDebug`, потому что печатает `QString` без лишних кавычек.
- `for (int port = ...; port <= portTo; ++port)` — обычный **цикл** (Урок 3.5!). Перебираем порты по очереди.
- Внутри — наш «стук» из Части 2.
- `QTcpSocket socket;` создаётся **внутри** цикла и сам уничтожается в конце каждого витка — это **RAII** (Часть D углублённого файла): трубка кладётся автоматически, мы не управляем ею вручную.
- `out.flush()` — «вытолкни на экран прямо сейчас», чтобы результаты появлялись по ходу, а не в самом конце.

🛠 Запусти на `127.0.0.1`. Если у тебя запущено что-то сетевое (например, веб-сервер или Qt-приложение-сервер), увидишь его порт как `OPEN`. На чистой системе открытых портов может не быть — это нормально.

⚠️ **Про скорость:** закрытые/«молчащие» порты ждут полные 200 мс. 1024 порта × 200 мс — это может быть долго. Поэтому:
- для теста бери диапазон поменьше (например `1..100`),
- или сканируй только **частые** порты (`80, 443, 445, 22, 3389`),
- меньше таймаут = быстрее, но можно «не услышать» медленный ответ.

🎯 **Попробуй сам:**
1. Поменяй диапазон на `1..100`, запусти.
2. Сделай так, чтобы программа печатала и закрытые порты тоже (строкой `zakryt`). Почувствуй разницу в выводе.
3. Сделай массив `int chastye[] = {22, 80, 443, 445, 3389};` и просканируй только их (через range-based for).

# Часть 4. 🛠 Сканер компьютеров в сети (рабочая программа)

🤖 Гениальная мысль: **сканер компьютеров — это тот же сканер портов**, только наоборот. Раньше мы держали **один компьютер** и меняли **порты**. Теперь будем держать **один порт** и менять **компьютеры** (IP-адреса). Если кто-то ответил на «стук» в общий порт — значит, компьютер **жив**.

**Шаг 1. Узнай свою подсеть.**
Открой в Qt Creator терминал (или меню `! команда` в Claude) и набери:
```
ipconfig
```
Найди строчку **IPv4-адрес**, например `192.168.1.37`. Значит твоя подсеть — `192.168.1.` (адреса от `192.168.1.1` до `192.168.1.254`). У тебя цифры могут быть другими (часто `192.168.0.` или `192.168.31.`).

**Шаг 2. Код.** Полный `main.cpp`:

```cpp
#include <QCoreApplication>
#include <QTcpSocket>
#include <QTextStream>

int main(int argc, char *argv[])
{
    QCoreApplication app(argc, argv);
    QTextStream out(stdout);

    QString subnet  = "192.168.1.";   // ⚠️ подставь СВОЮ подсеть (см. ipconfig)
    int     probe   = 445;            // порт-«пробник» (часто открыт у Windows)

    out << "Searching computers in network " << subnet << "1-254 ...\n";
    out.flush();

    for (int i = 1; i <= 254; ++i) {
        QString ip = subnet + QString::number(i);   // склеиваем "192.168.1." + "37"

        QTcpSocket socket;
        socket.connectToHost(ip, probe);

        if (socket.waitForConnected(150)) {
            out << "  found computer: " << ip << "\n";
            out.flush();
            socket.disconnectFromHost();
        }
    }

    out << "Search finished.\n";
    return 0;
}
```

🤖 Что нового:
- `QString::number(i)` — превращает число `37` в строку `"37"`, чтобы склеить полный адрес `"192.168.1.37"`.
- Снова цикл + «стук» — те же кубики, что и в сканере портов.

⚠️ **Честная оговорка:** этот способ находит только те компьютеры, у которых **открыт порт-пробник** (`445`). Если у компьютера всё закрыто (или он Mac/телефон) — он не ответит, хотя он есть. Профессиональные сканеры пробуют несколько способов сразу. Наш — простой и понятный.

🤖 Классический другой способ — «пинговать» (отправлять ICMP-эхо). В Qt для этого обычно запускают системную команду `ping` через класс `QProcess`. Это чуть сложнее, разберём, когда будем дорабатывать.

🎯 **Попробуй сам:**
1. Подставь свою подсеть (из `ipconfig`) и запусти. Найди в списке свой роутер (обычно `...1`).
2. Поменяй порт-пробник на `80` — изменился ли список найденных?
3. Объедини оба сканера: для **каждого** найденного компьютера сразу просканируй его частые порты.

# Часть 5. Делаем инструмент командной строки (CLI) ⌨️

> До сих пор адрес и порты были **зашиты в код** — чтобы поменять цель, надо лезть в `main.cpp` и пересобирать. Настоящий инструмент берёт всё из **командной строки**: `portscan --host 127.0.0.1 --from 1 --to 1024`. А ещё его можно вызывать **из любого места**, добавив в системный путь (PATH). Сделаем это и для **Windows**, и для **Mac**.

🛑 Напоминание: инструмент станет удобнее запускать — тем более держим правило из начала файла: сканируем **только своё**.

## Урок 5.1. Аргументы командной строки: `argc` и `argv`

🧠 Когда ты вводишь в консоли `portscan 127.0.0.1 80`, операционная система передаёт эти слова прямо в `main`:

```cpp
int main(int argc, char* argv[]) {
    // argc — сколько слов (включая имя программы)
    // argv — массив этих слов:
    //   argv[0] = "portscan"    (имя самой программы)
    //   argv[1] = "127.0.0.1"
    //   argv[2] = "80"
}
```

🕵️ **Под капотом:** этот массив кладёт в память **загрузчик ОС** при запуске процесса (помнишь рождение процесса в файле «Под капотом»?). Программа просто читает готовое.

🤖 Разбирать `argv` руками можно, но скучно и легко ошибиться. В Qt есть удобный помощник — **`QCommandLineParser`**.

## Урок 5.2. `QCommandLineParser` — параметры по-взрослому

🧠 Этот класс сам разбирает `--host`, короткие `-H`, значения по умолчанию и даже **бесплатно даёт `--help` и `--version`**. Вот сканер портов, управляемый из командной строки:

```cpp
#include <QCoreApplication>
#include <QCommandLineParser>
#include <QTcpSocket>
#include <QTextStream>

int main(int argc, char *argv[])
{
    QCoreApplication app(argc, argv);
    QCoreApplication::setApplicationName("portscan");
    QCoreApplication::setApplicationVersion("1.0");

    QCommandLineParser parser;
    parser.setApplicationDescription("Simple port scanner (your own networks only!)");
    parser.addHelpOption();       // даёт --help / -h
    parser.addVersionOption();    // даёт --version / -v

    //                     имена опции          описание            имя значения  по умолч.
    QCommandLineOption hostOpt({"H", "host"},    "Target address.",   "host", "127.0.0.1");
    QCommandLineOption fromOpt({"f", "from"},    "Start port.",       "port", "1");
    QCommandLineOption toOpt(  {"t", "to"},      "End port.",         "port", "1024");
    QCommandLineOption timeoutOpt("timeout",     "Per-port timeout, ms.", "ms", "200");

    parser.addOption(hostOpt);
    parser.addOption(fromOpt);
    parser.addOption(toOpt);
    parser.addOption(timeoutOpt);

    parser.process(app);          // разобрать argv (сам обработает --help и ошибки)

    QString host    = parser.value(hostOpt);
    int     from    = parser.value(fromOpt).toInt();
    int     to      = parser.value(toOpt).toInt();
    int     timeout = parser.value(timeoutOpt).toInt();

    QTextStream out(stdout);
    out << "Scanning " << host << " ports " << from << ".." << to << "\n";
    out.flush();

    for (int port = from; port <= to; ++port) {
        QTcpSocket socket;
        socket.connectToHost(host, port);
        if (socket.waitForConnected(timeout)) {
            out << "  port " << port << " OPEN\n";
            out.flush();
            socket.disconnectFromHost();
        }
    }
    out << "Done.\n";
    return 0;
}
```

🤖 Разбор:
- `QCommandLineOption({"H","host"}, описание, имя_значения, по_умолчанию)` — описываем одну опцию: длинное имя `--host`, короткое `-H`, значение по умолчанию.
- `parser.process(app)` — разбирает всё; если ввели `--help`, сам покажет справку и выйдет.
- `parser.value(opt).toInt()` — берём значение (или умолчание) и превращаем в число.

⚠️ Короткое `-h` уже занято справкой, поэтому для host берём `-H` (заглавная). У `--timeout` я не делал короткой формы, чтобы не путать с `-t` (`to`).

🛠 **`.pro`:** ничего нового — `QCommandLineParser` лежит в модуле `core`, который уже подключён.

Теперь программой можно пользоваться так:
```
portscan --help
portscan --host 127.0.0.1 --from 1 --to 100
portscan -H 192.168.1.1 -f 1 -t 1024 --timeout 300
```

🎯 **Попробуй сам:** добавь опцию `--closed`, чтобы печатать и закрытые порты. Потом сделай такой же CLI для сканера сети (опции `--net` и `--port`).

## Урок 5.3. ⚠️ Важно: Qt-программе нужны её библиотеки

🧠 Прежде чем «ставить» инструмент в систему — одна важная вещь. Твой `portscan` использует Qt, а значит **зависит** от библиотек Qt (`Qt6Core`, `Qt6Network`) и runtime компилятора. Внутри Qt Creator они находятся сами. Но если просто скопировать программу в другое место и запустить — будет ошибка вроде «не найден `Qt6Core.dll`».

Решение — положить рядом с программой все нужные библиотеки. Для этого есть готовые инструменты:
- **Windows:** `windeployqt`
- **Mac:** `macdeployqt`

Дальше — по шагам для каждой системы.

## Урок 5.4. 🪟 Windows: установить в PATH

🛠 По шагам:

1. **Собери в режиме Release.** Слева внизу в Qt Creator переключи `Debug → Release`, нажми `Ctrl+B`. Найди `portscan.exe` в папке сборки (вида `build-...-Release`).
2. **Сделай папку для инструмента**, например `C:\Tools\portscan`, и скопируй туда `portscan.exe`.
3. **Собери зависимости.** Открой из меню Пуск «Qt 6.x (MinGW) command prompt» (или найди `windeployqt.exe` в `C:\Qt\6.x.x\mingw_64\bin`) и выполни:
   ```
   windeployqt C:\Tools\portscan\portscan.exe
   ```
   Он скопирует нужные DLL рядом. Теперь программа запустится в любом месте.
4. **Добавь папку в PATH** — список папок, где система ищет программы по имени. Простой способ (PowerShell, для тебя, навсегда):
   ```powershell
   [Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Tools\portscan", "User")
   ```
   Или через окно: «Изменение переменных среды текущего пользователя» → переменная **Path** → «Создать» → вписать `C:\Tools\portscan`.
5. **Открой НОВОЕ окно консоли** (PATH применяется только в новых окнах) и проверь:
   ```
   portscan --host 127.0.0.1 --from 1 --to 100
   ```

🤖 Что такое **PATH**: когда ты вводишь команду, система ищет такую программу по папкам из списка PATH. Добавил свою папку → можешь звать программу по имени откуда угодно.

## Урок 5.5. 🍎 Mac: установить в PATH

🧠 На Mac наша `.pro` уже без `app_bundle` (строка `CONFIG -= app_bundle`), поэтому получается **обычный бинарник** `portscan`, а не `.app`. macOS использует оболочку **zsh**.

🛠 По шагам:

1. **Собери в Release**, найди бинарник `portscan` в папке сборки.
2. **Зависимости Qt.** Чтобы бинарник нашёл фреймворки Qt вне Qt Creator, проще всего поставить Qt через **Homebrew** (`brew install qt`) — тогда библиотеки лежат в стандартном месте. (Для «коробочной» установки Qt используют `macdeployqt` с .app-обёрткой — это разберём, когда будем дорабатывать.)
3. **Сделай файл исполняемым** (обычно уже так):
   ```
   chmod +x portscan
   ```
4. **Добавь в PATH.** Два простых варианта:
   - **Самый лёгкий** — скопировать в `/usr/local/bin` (эта папка почти всегда уже в PATH):
     ```
     cp portscan /usr/local/bin/portscan
     ```
     (если попросит права — добавь `sudo` спереди)
   - **Или** держать в своей папке `~/tools` и добавить её в PATH. Открой файл `~/.zshrc` и допиши строку:
     ```
     export PATH="$HOME/tools:$PATH"
     ```
     затем выполни `source ~/.zshrc` (или просто открой новое окно терминала).
5. **Проверь в новом окне терминала:**
   ```
   portscan --host 127.0.0.1 --from 1 --to 100
   ```

🤖 Если macOS заругается, что бинарник «от неизвестного разработчика» — для своей собственной программы это снимается командой `xattr -d com.apple.quarantine portscan` (или правым кликом → «Открыть»).

## Урок 5.6. Шпаргалка: Windows ⟷ Mac

| Шаг | Windows 🪟 | Mac 🍎 |

# Часть 6. Как сделать «по-взрослому» (следующий уровень) 🚀

🤖 Наши сканеры простые и **медленные**, потому что стучатся **по очереди** (один за другим). Вот куда расти:

- ⚡ **Скорость — асинхронность.** Вместо `waitForConnected` (который замораживает программу) использовать **сигналы и слоты** (`QTcpSocket::connected` / `errorOccurred`) и запускать **сотни** «стуков» одновременно. Тогда сканирование сети занимает секунды, а не минуты. (Привет Части 4 файла про Qt — сигналы и слоты!)
- 🪟 **Окно вместо консоли.** Сделать GUI: поле `QLineEdit` для цели, кнопку `QPushButton` «Сканировать», и `QTextEdit` для результатов. По нажатию кнопки — запуск (через сигнал/слот).
- 🏷️ **Имена компьютеров.** Класс `QHostInfo` умеет узнавать имя по IP (и наоборот).
- 🗄️ **Сохранение результатов.** Складывать находки в базу SQLite (`QtSql`) или в файл — чтобы сравнивать «что было вчера, что сегодня». (Привет Части 7 файла про Qt.)
- 🔎 **Точнее статусы.** Различать «закрыт» и «отфильтрован файрволом» по типу ошибки сокета.

✅ **Правило Боба:** сначала сделай **простую работающую** версию (как у нас), убедись, что понимаешь каждую строчку, и только потом улучшай. Рабочее-и-понятное лучше, чем сложное-и-сломанное.
