# 🤖👉 Курс C++ с Бобом — Указатели и ссылки

> Две важные, но пугающие новичков темы — **указатель** (`*`) и **ссылка** (`&`). На самом деле идея простая. Разберём подробно, с маленькими запускаемыми примерами и техническими аналогиями: что это, чем отличаются, и **что когда выбирать**.

**Когда читать:** после README (Урок 4.4–4.5 — адреса и память) и `Курс_C++_Основы_глубже.md` (Часть A — ссылки против копий). Связано со `Курс_C++_Стек_и_куча.md` и RAII.

Значки: 🤖 объяснение · 🧠 теория · 🛠 практика · 🎯 задание · ⚠️ важно · ✅ правило · 🕵️ реверс · 🔧 техника.

🧠 **Главная мысль одной строкой:**
- **Указатель** — это **переменная, которая хранит адрес** другой переменной (отдельный объект; может быть пустым и менять, на что смотрит).
- **Ссылка** — это просто **второе имя** уже существующей переменной (не отдельный объект; не бывает пустой и не меняет цель).

# Часть 1. Адрес и указатель 🏠

## Урок 1.1. Адрес: `&`

🧠 Память — это длинная улица из ячеек, у каждой свой **адрес** (номер дома). Знак `&` перед переменной даёт её адрес.

```cpp
#include <iostream>

int main() {
    int x = 42;
    std::cout << x  << "\n";   // 42         — значение (что в ящике)
    std::cout << &x << "\n";   // 0x61fe1c   — адрес (где ящик стоит)
}
```

🔧 Аналогия: `x` — это ящик с числом, а `&x` — **адрес склада**, где этот ящик стоит.

## Урок 1.2. Указатель: переменная с адресом

🧠 **Указатель** — это переменная, в которой лежит **адрес**. Тип пишут со звёздочкой: `int*` — «указатель на int».

```cpp
#include <iostream>

int main() {
    int x = 42;
    int* p = &x;        // p хранит АДРЕС x

    std::cout << p  << "\n";   // адрес x
    std::cout << *p << "\n";   // 42 — «разыменование»: сходить по адресу и взять значение

    *p = 100;           // изменить x ЧЕРЕЗ указатель
    std::cout << x  << "\n";   // 100 ! — мы поменяли x, ни разу не написав x =
}
```

🤖 Два главных значка:
- `&x` — «дай адрес x» (узнать, где ящик);
- `*p` — «разыменовать»: сходить по адресу из `p` и поработать с тем, что там лежит (открыть ящик).

🔧 Аналогия: указатель `p` — это **записка, на которой написан адрес** склада. `*p` — «иди по адресу с записки и открой ящик». Записку можно передать другу, переписать на ней другой адрес или оставить пустой.

## Урок 1.3. Пустой указатель: `nullptr`

🧠 Указатель может **никуда** не указывать — тогда в нём `nullptr` (пустая записка).

```cpp
int* p = nullptr;          // адрес не записан
if (p != nullptr) {
    std::cout << *p;       // зайдём, только если есть куда
}
// *p;  // ⛔ КРАХ: разыменование nullptr — идём по пустому адресу
```

⚠️ Разыменовать `nullptr` — **падение программы**. Поэтому перед использованием «чужого» указателя часто проверяют, что он не пустой.

# Часть 2. Ссылка — второе имя 🏷️

🧠 **Ссылка** (`&` в объявлении) — это **псевдоним**, второе имя той же переменной. Не новый ящик, а **вторая табличка** на том же ящике.

```cpp
#include <iostream>

int main() {
    int x = 5;
    int& r = x;         // r — ВТОРОЕ ИМЯ для x (не копия!)

    r = 99;             // меняем через r — без всякой звёздочки
    std::cout << x << "\n";   // 99 — ведь r и x это одно и то же
}
```

🔧 Аналогия: к станку прикрутили **вторую табличку с именем**. Имя другое, станок тот же. И табличку **нельзя переставить** на другой станок — она приварена.

⚠️ Две особенности ссылки:
- её **обязательно** сразу привязать (`int& r;` без `= x` — ошибка);
- её **нельзя перенаправить** на другую переменную потом.

```cpp
int a = 1, b = 2;
int& r = a;
r = b;        // ⚠️ это НЕ «перенаправить r на b»! Это «положить значение b в a».
              // r по-прежнему второе имя для a; теперь a == 2.
```

🤖 Это частая ловушка: `r = b` у ссылки — это **присваивание значения**, а не смена цели. У ссылки цель навсегда одна.

# Часть 3. Указатель против ссылки ⚔️

🧠 Главная таблица темы:

| | Указатель `int* p` | Ссылка `int& r` |
|---|---|---|
| Что это | переменная с **адресом** (отдельный объект) | **второе имя** той же переменной |
| Может быть пустым (`nullptr`)? | **да** | **нет**, всегда на что-то указывает |
| Можно перенаправить? | **да** (записать другой адрес) | **нет**, цель навсегда |
| Обязательно сразу задать? | нет (но лучше) | **да** |
| Как достучаться до значения | `*p`, поля — через `p->` | как к обычной переменной |
| Аналогия | записка с адресом (можно переписать/стереть) | приваренная вторая табличка |

🤖 Грубое правило: **ссылка** — когда «всегда есть и не меняется» (проще и безопаснее). **Указатель** — когда нужно «может быть пусто» или «менять, на что смотрю».

# Часть 4. Передача в функции: копия / указатель / ссылка ⚙️

🧠 Самое полезное на практике — как функция меняет (или не меняет) оригинал.

```cpp
#include <iostream>

void byValue(int n)     { n = 100; }   // копия — оригинал не тронут
void byPointer(int* n)  { *n = 100; }  // меняем оригинал через адрес
void byReference(int& n){ n = 100; }   // меняем оригинал через второе имя

int main() {
    int x = 5;

    byValue(x);       std::cout << x << "\n";   // 5   (меняли копию)
    byPointer(&x);    std::cout << x << "\n";   // 100 (передали адрес)
    byReference(x);   std::cout << x << "\n";   // 100 (передали ссылку)
}
```

🤖 Разница в вызове:
- `byValue(x)` — отдали **копию** значения, оригинал в безопасности;
- `byPointer(&x)` — отдали **адрес** (видно `&` на месте вызова), функция меняет оригинал;
- `byReference(x)` — отдали **ссылку**, выглядит как обычная передача, но меняет оригинал.

✅ **Правило Боба:** надо **изменить** оригинал — бери **ссылку** (`int&`), это чище указателя. Большие данные только **читаешь** — бери **`const`-ссылку** (`const std::string&`): без копии и без права испортить (помнишь Правило №3 из «Основ глубже»?).

```cpp
void print(const std::string& s) {   // не копируем (быстро) и не меняем (безопасно)
    std::cout << s << "\n";
}
```

# Часть 5. Стрелочка `->` для объектов

🧠 Если указатель смотрит на **структуру/объект**, к его полям/методам идут через `->` (это короткая запись для `(*p).field`).

```cpp
#include <iostream>

struct Point { int x; int y; };

int main() {
    Point pt{3, 5};
    Point* pp = &pt;

    std::cout << pp->x << "\n";       // 3   — стрелочка идёт к полю
    std::cout << (*pp).y << "\n";     // 5   — то же самое, но длиннее
    pp->x = 10;                       // меняем поле через указатель
    std::cout << pt.x << "\n";        // 10
}
```

🤖 Запомни: у объекта-**значения** — точка (`pt.x`), у **указателя** на объект — стрелочка (`pp->x`). Это та же `.`/`->` разница, что и с умными указателями.

# Часть 6. Опасности: пустой и висячий указатель 🔓

🧠 Указатель силён, но требует аккуратности. Два классических бага:

**1. Разыменование `nullptr`** (Урок 1.3) → крах. Проверяй перед использованием.

**2. Висячий указатель** — указывает на объект, который **уже умер**:

```cpp
int* bad() {
    int local = 5;
    return &local;     // ⚠️ local умрёт в конце функции — адрес станет «битым»
}
// тот, кто разыменует возвращённый указатель, получит мусор или крах
```

🔧 Аналогия: записка с адресом дома, который **уже снесли**. Пойдёшь по адресу — а там пусто. (Подробнее про это — в `Курс_C++_Стек_и_куча.md`.)

🕵️ **Безопасность:** разыменование `nullptr` и «use-after-free» (использование после смерти) — целые классы уязвимостей, которые ищут и атакующие, и защитники.

# Часть 7. `const` с указателями и ссылками (коротко)

🧠 `const` ограничивает, что можно менять:

```cpp
int x = 1, y = 2;

const int* p1 = &x;   // указатель на const: НЕЛЬЗЯ менять *p1, но можно p1 = &y
int* const p2 = &x;   // const-указатель: можно менять *p2, но НЕЛЬЗЯ p2 = &y

const std::string& r = name;   // ссылка на const — только читать (самый частый случай)
```

🤖 Для новичка важнее всего одно: **`const`-ссылка** (`const T&`) — стандартный способ передать что-то большое в функцию «только посмотреть». Остальные тонкости `const`-указателей — на потом.

# Часть 8. Что выбрать (и связь с умными указателями) 🧭

✅ **Правила Боба:**

1. **По умолчанию — ссылка.** Если объект всегда есть и не меняется цель — ссылка чище и безопаснее.
2. **Указатель** — когда нужно: «может быть пусто» (`nullptr`), «менять, на что смотрю», пройти по массиву, или работать с API ОС/C.
3. **Сырой указатель `T*` — это «просто посмотреть»** (наблюдатель), он **не владеет** объектом.
4. **Владение — через умные указатели**, а не сырые `new`/`delete`: `std::unique_ptr` / `std::shared_ptr` (см. RAII-файлы). Сырой `new` для владения в современном C++ не пишут.

🤖 Связь: умный указатель — это «сырой указатель, который сам убирает за собой» (RAII). Поэтому, поняв обычные указатели, ты уже наполовину понял умные.

# 🕵️ Указатели, ссылки и реверс

- **Указатель = адрес**, поэтому реверсеры постоянно с ними работают: `*p` и `p->field` превращаются в инструкции «загрузи из памяти по адресу».
- **Ссылка** под капотом обычно реализована тем же адресом — на уровне ассемблера ссылка и указатель часто выглядят одинаково (но в C++ ведут себя по-разному и защищают тебя от ошибок).
- Разыменование `nullptr` → характерный аварийный сбой; «use-after-free» по висячему указателю → лакомая цель для эксплойтов. Понимая это, ты понимаешь и как защищаться.

## 📋 Шпаргалка

| Запись | Смысл |
|---|---|
| `&x` | адрес переменной `x` |
| `int* p = &x;` | указатель `p` хранит адрес `x` |
| `*p` | разыменование: значение по адресу из `p` |
| `p->field` | поле объекта через указатель (`(*p).field`) |
| `nullptr` | «пустой» указатель (никуда) |
| `int& r = x;` | ссылка: второе имя для `x` |
| `const T&` | ссылка «только читать» (передать без копии) |

## 📖 Словарик

| Слово | Простыми словами |
|---|---|
| **Адрес** | Номер ячейки памяти, где лежит переменная |
| **Указатель** | Переменная, хранящая адрес (записка с адресом) |
| **Разыменование (`*p`)** | Сходить по адресу и взять/положить значение |
| **`&` (адрес-оф)** | Получить адрес переменной |
| **`nullptr`** | Пустой указатель, никуда не указывает |
| **Ссылка** | Второе имя существующей переменной (псевдоним) |
| **`const`-ссылка** | Псевдоним «только для чтения» |
| **Висячий указатель** | Указывает на уже уничтоженный объект |
| **`->`** | Доступ к полю объекта через указатель |

---

> 🤖 Главное: **указатель** — переменная-«записка» с адресом (можно пустую, можно переписать; `*p`/`->`); **ссылка** — второе имя той же переменной (всегда валидна, не меняется). Менять оригинал в функции — ссылка; «может быть пусто/гулять» — указатель; **владеть** — умный указатель.
>
> Будем **дорабатывать**: указатели на функции, массивы и арифметика указателей, `std::reference_wrapper`.
