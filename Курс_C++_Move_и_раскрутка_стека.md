# 🤖⚙️ Курс C++ с Бобом — Раскрутка стека, std::move и нулевая стоимость

> Три механизма «под капотом», благодаря которым C++ остаётся **и безопасным, и быстрым**:
> - **раскрутка стека** — что происходит при ошибке и почему RAII спасает;
> - **`std::move`** — передавать вещи вместо копирования (быстро);
> - **нулевая стоимость** (*zero-cost abstractions*) — почему все эти удобства **не замедляют** программу.

# Часть 1. Раскрутка стека (stack unwinding) 🌀

## Урок 1.1. Что это такое

🧠 Помнишь стек вызовов (из «Под капотом»)? Функции вызывают друг друга, и у каждой свой «кадр» с локальными объектами: `main` → `rabota` → `glubzhe` …

Теперь представь, что глубоко внутри случилась **ошибка** и программа «кинула исключение» (`throw` — крикнула «авария!»). Программа бросает обычную работу и начинает **карабкаться вверх по стеку** обратно к тому, кто умеет ошибку обработать (`catch`). И вот самое важное: **поднимаясь, она уничтожает все локальные объекты** на своём пути — вызывает их **деструкторы**. Этот «подъём с уборкой» и называется **раскрутка стека**.

🔧 **Аналогия:** ты поднялся по этажам здания (каждая функция — этаж, на каждом что-то включил). Сработала пожарная тревога (исключение). Ты спускаешься этаж за этажом к выходу (`catch`), и на **каждом этаже автоматически выключается** всё, что ты включал (деструкторы). Дошёл до выхода — ничего не осталось включённым. Само слово «раскрутка» = «разматывание стека обратно, как катушки».

## Урок 1.2. Проверим живьём

🤖 Тут понадобятся три новых слова (подробно про них — отдельная тема, сейчас только ради примера):
- `throw` — «кинуть» сигнал ошибки;
- `try { ... }` — «попробуй сделать»;
- `catch (...)` — «поймай ошибку, если случится».

```cpp
#include <iostream>
#include <string>
#include <stdexcept>

struct Resurs {
    std::string imya;
    Resurs(std::string n) : imya{n} { std::cout << imya << " vzyat\n"; }
    ~Resurs() { std::cout << imya << " osvobozhdyon\n"; }   // сработает при раскрутке!
};

void glubzhe() {
    Resurs r{"B"};
    std::cout << "brosaem oshibku!\n";
    throw std::runtime_error("polomka");          // кидаем исключение
    std::cout << "eto ne vypolnitsya\n";          // сюда уже не дойдём
}

void rabota() {
    Resurs r{"A"};
    glubzhe();
    std::cout << "i eto ne vypolnitsya\n";
}

int main() {
    try {
        rabota();
    } catch (const std::exception& e) {
        std::cout << "poymali: " << e.what() << "\n";
    }
}
```

**Ожидаемый вывод:**
```
A vzyat
B vzyat
brosaem oshibku!
B osvobozhdyon
A osvobozhdyon
poymali: polomka
```

🤖 Смотри, что произошло после `throw`:
- сначала уничтожился `B` (вышли из `glubzhe`) → `B osvobozhdyon`;
- потом `A` (вышли из `rabota`) → `A osvobozhdyon`;
- и только потом ошибку поймал `catch` в `main`.

**Это и есть раскрутка стека.** Вот почему в RAII-файле мы говорили: деструктор срабатывает **даже при ошибке** — потому что при раскрутке он вызывается автоматически. RAII и раскрутка стека — лучшие друзья.

⚠️ Поэтому же деструктор **не должен сам бросать исключения**: если он кинет ошибку во время раскрутки (когда уже разбирается другая ошибка) — программа аварийно завершится.

# Часть 2. `std::move` — передать, а не копировать 🚚

## Урок 2.1. Копия дорогая, передача дешёвая

🧠 Копировать большие объекты **дорого**: например, скопировать вектор на миллион чисел — значит продублировать миллион чисел. Но часто оригинал тебе **больше не нужен** — ты хочешь просто **передать** содержимое дальше. Для этого есть **перемещение** (*move*).

🔧 **Аналогия:**
- **Копирование** = собрать на заводе **вторую такую же** машину — долго и дорого.
- **Перемещение (move)** = отдать **ключи и ПТС** от уже готовой машины новому владельцу. Машина та же, просто сменила хозяина. **Мгновенно.**

🤖 `std::move` сам по себе ничего не «двигает» — это **пометка**: «этот объект мне больше не нужен, можешь забрать его содержимое (а не копировать)».

## Урок 2.2. Пример с вектором

```cpp
#include <iostream>
#include <vector>
#include <utility>     // std::move

int main() {
    std::vector<int> a = {1, 2, 3, 4, 5};

    std::vector<int> b = std::move(a);     // передаём содержимое a в b (НЕ копируем)

    std::cout << "b size: " << b.size() << "\n";   // 5
    std::cout << "a size: " << a.size() << "\n";    // 0 — у a забрали содержимое
}
```

**Ожидаемый вывод:**
```
b size: 5
a size: 0
```

🤖 `b` забрал внутренний буфер `a` — без копирования элементов (просто «передали ПТС»). После этого `a` остаётся пустым (обычно `size() == 0`).

⚠️ **Главное правило:** после `std::move(a)` **не используй старое значение `a`** — ты ведь отдал ПТС, машина больше не твоя. Присвоить `a` что-то новое можно, а вот рассчитывать на его прежнее содержимое — нельзя.

## Урок 2.3. Move и владение (связь с RAII)

🧠 Помнишь, в RAII-файле мы **запрещали копировать** владельца ресурса (`= delete`)? `move` — это **правильный способ всё же передать** ресурс другому: не копией, а перемещением. Так устроен `unique_ptr`:

```cpp
#include <memory>
auto p1 = std::make_unique<int>(42);
auto p2 = std::move(p1);     // передаём ВЛАДЕНИЕ от p1 к p2
// теперь p2 владеет числом, а p1 == nullptr (пустой)
```

🤖 Скопировать единственного хозяина нельзя (было бы два владельца одного ресурса — двойное освобождение). А **передать** — можно, через `move`. Поэтому `unique_ptr` называют «move-only».

🤖 Часто move случается **сам**: например, когда функция **возвращает** локальный объект — компилятор перемещает его, а не копирует. Так что ты получаешь скорость move, даже не написав `std::move`.

✅ **Правило Боба:** копировать большое — дорого. Если оригинал больше не нужен — **передавай через `move`**. После `move` старый объект не используй.

# Часть 3. lvalue и rvalue — «именованное» и «временное» 🏷️

> Странные слова, но идея простая — и именно она объясняет, **почему `std::move` вообще работает**. Берём по делу.

## Урок 3.1. Две породы выражений

🧠 Любое выражение в C++ — это одна из двух «пород»:

- **lvalue** («эл-вэлью») — у него есть **имя и место** (адрес). Оно **остаётся жить** после строки. Можно взять его адрес через `&`.
- **rvalue** («ар-вэлью») — это **временное** значение **без имени**. Оно живёт «одну секунду» — посчиталось и сейчас исчезнет. Адрес у него взять нельзя.

🤖 Откуда буквы: **l** — *left* (раньше такие стояли **слева** от `=`), **r** — *right* (справа). Грубое правило-подсказка: *«можешь поставить слева от `=` и взять адрес — lvalue; временное, без имени — rvalue»*.

```cpp
int x = 10;        // x — LVALUE: есть имя, есть адрес, живёт дальше
int y = x + 5;     // x+5 — RVALUE: временный результат, имени нет, тут же исчезнет

int* p = &x;       // ✅ у lvalue есть адрес — можно
// int* q = &(x + 5);  // ⛔ ОШИБКА: у rvalue адреса нет
x = 99;            // ✅ lvalue можно слева от =
// (x + 5) = 99;   // ⛔ ОШИБКА: rvalue слева от = нельзя
```

🔧 **Техническая аналогия:**
- **lvalue** — машина **с номером, стоящая в гараже**: у неё есть «адрес», к ней можно вернуться, она никуда не денется.
- **rvalue** — деталь, **только что сошедшая с конвейера**, ещё без номера, которую сейчас же используют. Она временная — поэтому её содержимое можно **смело забрать**, никто больше за ней не придёт.

Примеры rvalue (временных): `x + 5`, `2 * 3`, `"текст"`, результат функции `summa(2,3)`, `std::string("Bob")`.
Примеры lvalue (именованных): переменная `x`, `imya`, элемент `v[0]`, поле `obj.pole`.

## Урок 3.2. Зачем это нужно: вот тут и появляется move ⚡

🧠 Главная мысль всего раздела:

> 🤖 У **rvalue** (временного) содержимое можно **забрать** (move) — ведь оно всё равно сейчас исчезнет, никто не заметит. А у **lvalue** (именованного) забирать **опасно** — его ещё будут использовать дальше, надо **копировать**.

Вот почему компилятор различает породы: чтобы для временных делать **быстрый move**, а для именованных — **безопасную копию**. Проверим живьём через две версии функции:

```cpp
#include <iostream>
#include <string>

void obrabotat(const std::string& s) {       // ловит LVALUE (именованное)
    std::cout << "kopiruem (lvalue): " << s << "\n";
}
void obrabotat(std::string&& s) {             // ловит RVALUE (временное) — двойной &&
    std::cout << "zabiraem/move (rvalue): " << s << "\n";
}

int main() {
    std::string imya = "Bob";

    obrabotat(imya);              // imya — LVALUE -> копируем (оригинал ещё нужен)
    obrabotat("vremennyy");       // "..." — RVALUE -> забираем (временный)
    obrabotat(imya + "!");        // imya+"!" — RVALUE -> забираем
    obrabotat(std::move(imya));   // std::move превратил LVALUE в RVALUE -> забираем
}
```

**Ожидаемый вывод:**
```
kopiruem (lvalue): Bob
zabiraem/move (rvalue): vremennyy
zabiraem/move (rvalue): Bob!
zabiraem/move (rvalue): Bob
```

🤖 Разбор:
- `std::string&&` — это **rvalue-ссылка** (двойной амперсанд `&&`). Она цепляется **только за временные** значения. По ней компилятор и понимает: «это временное, можно забрать».
- `const std::string&` — обычная ссылка, ловит именованные (и вообще всё, что не подошло к `&&`).
- Компилятор **сам выбирает** нужную версию по породе аргумента. Это происходит на этапе сборки, бесплатно.

## Урок 3.3. Что же такое `std::move` на самом деле

🧠 Теперь точный ответ (в Части 2 мы сказали «пометка» — вот она подробно):

> **`std::move` ничего не перемещает.** Это просто **превращение** lvalue в rvalue — «расписка»: *«я обещаю, что больше не буду пользоваться этим объектом, можешь забрать его содержимое»*.

```cpp
std::string a = "dlinnaya stroka";
std::string b = a;              // a — lvalue -> КОПИЯ (a остаётся целой)
std::string c = std::move(a);   // мы пообещали не трогать a -> ПЕРЕМЕЩЕНИЕ (забрали содержимое)
// теперь a использовать нельзя (мы же обещали)
```

🔧 Аналогия: `std::move(a)` — это как **снять с машины номер и отдать ключи**: ты официально заявил «эта машина мне больше не нужна, забирай». Само «забирание» делает уже move-конструктор — а `std::move` лишь **дал разрешение**.

⚠️ Поэтому правило из Части 2 и обретает смысл: **после `std::move(a)` объект `a` не используй** — ты ведь обещал.

## Урок 3.4. Где это встречается на практике

🧠 Тебе не придётся писать `&&` каждый день, но понимать — важно, потому что это работает **само** повсюду:

- **`push_back`** в контейнерах сам выбирает: временный объект — переместит, именованный — скопирует.
  ```cpp
  std::vector<std::string> v;
  std::string s = "tekst";
  v.push_back(s);             // s — lvalue -> копия (s ещё нужна)
  v.push_back(std::move(s));  // обещали не трогать -> перемещение (быстро)
  v.push_back("srazu");       // rvalue -> перемещение само
  ```
- **возврат из функции** (`return bolshoy_vektor;`) — компилятор сам перемещает, не копирует.
- **`unique_ptr`** можно только перемещать (он rvalue-only по сути) — отсюда `auto p2 = std::move(p1);`.
- Свой **move-конструктор** класса принимает `&&` (как написать — разберём в «дорабатывать»).

✅ **Правило Боба:** **lvalue** — именованное, живёт дальше → **копируем**. **rvalue** — временное, сейчас исчезнет → можно **забрать (move)**. `std::move` — это «расписка», превращающая твоё именованное в «можно забрать»; после неё старое не трогай. В 99% случаев компилятор делает это **за тебя**.

🎯 **Попробуй сам:** сделай две функции `pokazhi(const std::string&)` и `pokazhi(std::string&&)` (как в Уроке 3.2) и прогони через них: переменную, строковый литерал, `a + b` и `std::move(переменная)`. Сверь, какая версия вызвалась.

# Часть 4. Нулевая стоимость (zero-cost abstractions) 🎯

## Урок 3.1. Главный принцип C++

🧠 У C++ есть знаменитое правило — **«абстракции с нулевой стоимостью»** (*zero-cost abstractions*, или *zero-overhead principle*). Оно из двух частей:

1. **Ты не платишь за то, чем не пользуешься.**
2. **А то, чем пользуешься, ты не смог бы написать руками быстрее.**

🤖 Простыми словами: все удобные и безопасные инструменты, что мы учили — **RAII, умные указатели, `move`, `for` по контейнеру, классы, шаблоны** — превращаются компилятором в машинный код **такой же быстрый**, как если бы ты писал низкоуровневый код руками. Удобство достаётся **бесплатно** по скорости.

🔧 **Аналогия:** это как удобная приборная панель в машине, которая при этом **ни на грамм не утяжеляет** её и **не замедляет**. Получаешь и удобство, и скорость — не выбирая.

## Урок 3.2. Примеры «бесплатных» абстракций

```cpp
// высокоуровнево и безопасно:
auto p = std::make_unique<int>(42);
*p = 100;

// компилятор превращает это примерно в:
//   int* p = new int(42);
//   *p = 100;
//   delete p;            // в конце, автоматически
// — РОВНО тот же машинный код, что ты написал бы руками. Бесплатно!
```

Так же «бесплатны»:
- `for (int x : v)` — компилируется как обычный цикл по индексам;
- `std::sort` — не медленнее аккуратно написанной вручную сортировки (обычно быстрее);
- методы класса — встраиваются (inline), как обычные функции;
- шаблоны — под каждый тип генерируется отдельный быстрый код.

## Урок 3.3. А что НЕ бесплатно?

🧠 Честно: кое-что всё же стоит немного — но ровно по принципу «платишь, только если используешь»:
- `virtual` — крошечный поиск в vtable (платишь, только когда нужен полиморфизм);
- `shared_ptr` — счётчик хозяев чуть дороже `unique_ptr`;
- исключения — стоят, только когда реально брошены.

🤖 Поэтому совет: бери `unique_ptr` по умолчанию (он бесплатный), а `shared_ptr`/`virtual` — когда они вправду нужны.

🕵️ **Реверс-связь:** из-за нулевой стоимости реверсер часто **не может** понять, писал ли ты `unique_ptr` или сырые `new`/`delete` — они превращаются в **одинаковые** инструкции. Абстракция исчезает на уровне машинного кода.

✅ **Правило Боба:** в C++ можно писать **высокоуровнево, безопасно и читаемо** и при этом получать **низкоуровневую скорость**. Поэтому на C++ делают игры, браузеры, биржи — там, где нужны сразу и надёжность, и скорость.

## 📋 Шпаргалка

| Тема | Суть |
|---|---|
| **Раскрутка стека** | При ошибке стек «разматывается» вверх, по пути срабатывают деструкторы локальных объектов → RAII убирает за собой даже при сбое |
| **lvalue / rvalue** | Именованное (живёт дальше → копируем) / временное (исчезнет → можно забрать=move) |
| **`std::move`** | Пометка «забери содержимое, не копируй»; быстро. После move старый объект не используй |
| **Нулевая стоимость** | Удобные абстракции компилируются в такой же быстрый код, как низкоуровневый; платишь только за то, что используешь |

## 📖 Словарик

| Слово | Простыми словами |
|---|---|
| **Исключение / `throw`** | Сигнал «случилась ошибка», брошенный вверх по стеку |
| **`try` / `catch`** | «Попробуй … / поймай ошибку, если будет» |
| **Раскрутка стека** | Подъём по стеку при ошибке с вызовом деструкторов |
| **`std::move`** | Пометка «можно забрать содержимое» (передать, не копировать) |
| **Перемещение (move)** | Передать ресурс другому объекту, не копируя |
| **lvalue** | Выражение с именем и адресом, живёт дальше (`x`, `v[0]`, `obj.pole`) |
| **rvalue** | Временное выражение без имени (`x+5`, `"text"`, `std::string("Bob")`) |
| **rvalue-ссылка `&&`** | Ссылка, цепляющаяся за временные; по ней видно «можно забрать» |
| **«Опустошённый» объект** | Тот, у кого забрали содержимое через move; не используй его |
| **Zero-cost / нулевая стоимость** | Абстракция компилируется без лишних затрат во время работы |

---

> 🤖 Главное тремя строками:
> - **раскрутка стека** — почему RAII спасает даже при ошибке (деструкторы срабатывают на пути вверх);
> - **`std::move`** — передать вместо копии (быстро), и после move старое не трогать;
