# Часть 1. `cout` — вывод подробно

🧠 `std::cout` (нужен `#include <iostream>`) печатает в консоль. Значки `<<` можно **сцеплять** в одну строку:

```cpp
int port = 80;
std::cout << "Port " << port << " is open\n";   // Port 80 is open
```

🤖 Полезные мелочи:
- **`"\n"` vs `std::endl`.** Оба переносят строку, но `std::endl` ещё и «проталкивает» вывод на экран сразу (*flush*) — это **медленнее**. По умолчанию пиши **`"\n"`**; `std::endl` бери, только когда нужно увидеть строку немедленно.
- **`bool` печатается как `1`/`0`**, а не `true`/`false` (это починим в Части 2).
- **Ошибки** лучше слать в `std::cerr` (отдельный поток ошибок), а не в `cout`.

```cpp
bool ok = true;
std::cout << ok << "\n";            // 1  (не "true"!)
std::cerr << "Error: no file\n";    // поток ошибок
```

# Часть 2. Форматированный вывод (манипуляторы) 🎛️

🧠 Чтобы вывод был **красивым** (ровные столбцы, нужное число знаков, hex), есть «манипуляторы» из `#include <iomanip>`.

### `boolalpha` — true/false вместо 1/0
```cpp
std::cout << std::boolalpha << true << " " << false << "\n";   // true false
```

### `fixed` + `setprecision` — знаки после запятой
```cpp
double pi = 3.14159265;
std::cout << pi << "\n";                                       // 3.14159
std::cout << std::fixed << std::setprecision(2) << pi << "\n"; // 3.14
```

### `hex` / `dec` — система счисления (привет реверсу 🕵️)
```cpp
int x = 255;
std::cout << std::hex << x << "\n";          // ff
std::cout << std::showbase << x << "\n";     // 0xff
std::cout << std::dec << x << "\n";          // 255 (вернули десятичную)
```
⚠️ `std::hex` **«липкий»**: остаётся включённым, пока не вернёшь `std::dec`.

### `setw` + `left`/`right` — ровные столбцы (таблица)
```cpp
std::cout << std::left << std::setw(8) << "Port" << std::setw(8) << "Status" << "\n";
std::cout << std::setw(8) << 80  << std::setw(8) << "open"   << "\n";
std::cout << std::setw(8) << 443 << std::setw(8) << "open"   << "\n";
std::cout << std::setw(8) << 22  << std::setw(8) << "closed" << "\n";
```

**Ожидаемый вывод:**
```
Port    Status  
80      open    
443     open    
22      closed  
```

🤖 `setw(8)` задаёт **ширину** следующего поля (8 символов), `left` — выравнивание по левому краю. Так получаются ровные колонки. (`setw` действует **только на следующий** вывод.)

# Часть 3. `std::format` — современный способ (C++20) ✨

🧠 Манипуляторы громоздкие. В C++20 появился `std::format` (`#include <format>`) с «дырками» `{}` — как в нормальных языках:

```cpp
std::cout << std::format("Port {} is {}\n", 80, "open");      // Port 80 is open
std::cout << std::format("pi = {:.2f}\n", 3.14159);           // pi = 3.14
std::cout << std::format("hex: {:#x}\n", 255);                // hex: 0xff
std::cout << std::format("{:<8}{:<8}\n", "Port", "Status");   // выравнивание по левому
std::cout << std::format("{:>6}\n", 42);                      // вправо, ширина 6
```

🤖 Спецификаторы внутри `{}`: `{:.2f}` — 2 знака после запятой, `{:#x}` — hex с `0x`, `{:<8}`/`{:>8}` — выравнивание и ширина. Это короче и понятнее манипуляторов. (Нужен **C++20** и свежий компилятор. 🔗 [Стандарты](Курс_C++_Стандарты.md).)

# Часть 4. `cin` — ввод подробно ⌨️

🧠 `std::cin >> x` читает **одно «слово»** (до пробела/переноса).

```cpp
int age;
std::cout << "Enter age: ";
std::cin >> age;                 // ввели 10 -> age = 10

int a, b;
std::cin >> a >> b;              // "3 5" -> a=3, b=5
```

### Прочитать целую строку — `std::getline`
🧠 `cin >>` останавливается на пробеле, поэтому имя из двух слов так не прочитать. Для **всей строки** — `std::getline`:

```cpp
std::string name;
std::getline(std::cin, name);    // читает строку целиком, вместе с пробелами
```

### ⚠️ Ловушка: `cin >>` перед `getline`
```cpp
int n;
std::cin >> n;                   // оставляет '\n' в буфере ввода
std::cin.ignore();               // ⚠️ «съесть» этот перенос строки
std::string line;
std::getline(std::cin, line);    // теперь читает правильно
```
🤖 После `cin >> n` в буфере остаётся непрочитанный перенос строки. Если сразу позвать `getline`, он прочитает **пустую** строку. Лечится `std::cin.ignore();`.

### Проверка плохого ввода
🧠 Если ждёшь число, а ввели буквы — `cin` **ломается** (failbit). Надо проверить и почистить:

```cpp
int port;
std::cout << "Enter port: ";
while (!(std::cin >> port)) {            // ввели не число?
    std::cin.clear();                    // сбросить флаг ошибки
    std::cin.ignore(10000, '\n');        // выкинуть плохой ввод
    std::cout << "Not a number, try again: ";
}
std::cout << "OK, port = " << port << "\n";
```

✅ **Правило Боба:** всегда будь готов к **неправильному вводу** — пользователь введёт что угодно. Проверяй `if (std::cin >> x)` и при ошибке делай `clear()` + `ignore()`.

# Часть 5. Qt-вывод: `qDebug` и `QTextStream` 🪟

🧠 В Qt у вывода своя история. Два главных инструмента — и у каждого своя роль.

### `qDebug()` — для отладочных сообщений
```cpp
#include <QDebug>
int port = 80;
qDebug() << "checking port" << port << "-> open";   // сам ставит пробелы и перенос строки
```

🤖 **Зачем нужен `qDebug`** (а не `cout`):
- `cout` **не умеет** печатать `QString` напрямую (надо `.toStdString()`), а `qDebug` умеет — и `QString`, и `QList`, и многие Qt-типы.
- В **оконной** программе консоли часто **нет**, и `cout` уходит «в никуда»; `qDebug` пишет в **окно отладки** (Application Output в Qt Creator) на любой системе.
- `qDebug` сам вставляет **пробелы** между значениями и **перенос строки** в конце.
- Это **отладочный** канал. Есть целое семейство по важности: `qDebug()` (отладка), `qInfo()` (инфо), `qWarning()` (предупреждение), `qCritical()` (серьёзная ошибка). Их можно **отключать**, не трогая код.

⚠️ Мелочь: `qDebug()` берёт строки в **кавычки** (`"text"`). Не нужны кавычки — `qDebug().noquote() << ...`.

### `QTextStream(stdout)` — чистый вывод в консоль
🧠 Когда нужен **обычный** вывод пользователю в консоль (как `cout`, но по-Qt и с поддержкой `QString`) — берут `QTextStream`:

```cpp
#include <QTextStream>
QTextStream out(stdout);
out << "Port " << 80 << " is open\n";    // чистый вывод, понимает QString
```

🔗 Именно `QTextStream(stdout)` мы использовали в [проекте-сканере](Курс_C++_Проект_Сканер.md) — для аккуратного вывода без «отладочного» оформления.

### Форматирование строк в Qt: `QString::arg`
```cpp
QString s = QString("Port %1 is %2").arg(80).arg("open");   // "Port 80 is open"
QString hex = QString("0x%1").arg(255, 0, 16);              // "0xff" (основание 16)
```
🤖 `%1`, `%2` — «дырки», `.arg(...)` подставляет значения по порядку. Это Qt-аналог `std::format`.

## 📋 Шпаргалка

| Хочу… | Делаю |
|---|---|
| Вывести значения | `std::cout << a << " " << b << "\n";` |
| Перенос строки | `"\n"` (а не медленный `std::endl`) |
| `true`/`false` | `std::cout << std::boolalpha << flag;` |
| 2 знака после запятой | `std::cout << std::fixed << std::setprecision(2);` |
| Hex | `std::cout << std::hex << x;` (вернуть: `std::dec`) |
| Ровные столбцы | `std::setw(n)` + `std::left` |
| Современно (C++20) | `std::format("{} {:.2f}", name, x)` |
| Прочитать число/слово | `std::cin >> x;` |
| Прочитать всю строку | `std::getline(std::cin, line);` |
| Защититься от плохого ввода | `clear()` + `ignore()` |
| Отладочный вывод (Qt) | `qDebug() << ...;` |
| Чистый вывод (Qt) | `QTextStream out(stdout); out << ...;` |

## 📖 Словарик

| Слово | Простыми словами |
|---|---|
| **`std::cout` / `std::cin`** | Вывод в консоль / ввод с клавиатуры |
| **`std::cerr`** | Поток для сообщений об ошибках |
| **Манипулятор** | Команда форматирования (`setw`, `hex`, `fixed`…) из `<iomanip>` |
| **flush** | «Протолкнуть» вывод на экран немедленно (`std::endl`) |
| **`std::getline`** | Прочитать целую строку (с пробелами) |
| **`std::format`** | Форматирование с `{}` (C++20) |
| **`qDebug`** | Отладочный вывод Qt (понимает `QString`, работает без консоли) |
| **`QTextStream`** | Чистый ввод/вывод по-Qt |
| **`QString::arg`** | Подстановка значений в `%1`, `%2`… |

---

> 🤖 Главное: **обычный C++** — `cout` для вывода (`"\n"`, не `endl`), `<iomanip>` или `std::format` для красоты, `cin`/`getline` для ввода (и проверяй плохой ввод!). **Qt** — `qDebug` для отладки (понимает `QString`, работает в окне без консоли), `QTextStream(stdout)` для чистого вывода, `QString::arg` для форматирования.
>
> Будем **дорабатывать**: чтение/запись **файлов** через потоки (`std::ifstream`/`ofstream`), и `std::format` подробнее.
